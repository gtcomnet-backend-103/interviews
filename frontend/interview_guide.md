# Frontend Engineer Interview Guide (Mid-to-Senior)

## Interview Overview
- **Duration**: 60-75 minutes
- **Format**: 1-on-1 Conversational System Design & Fundamentals
- **Core Stack Focus**: JavaScript/TypeScript, React, Vue, State Management, UI Architecture

---

## Interview Phases
The interview is split into the following sections. **No heavy live coding is required**; the focus is on architectural thinking, conversational logic, and design patterns.

1. `01-Introduction.md`
2. `02-Core_JS_TS_Fundamentals.md`
3. `03-React_Vue_Knowledge.md`
4. `04-Architecture_and_Design.md`
5. `05-Cultural_Fit_and_QA.md`

---

## Scoring Rubric

- **Junior Level (Red Flags):** Struggles to explain basic JS concepts (Event Loop, Closures). Wants to put all state into a global store. Suggests rendering all 10,000 rows in a table at once.
- **Mid-Level (Passing):** Solid understanding of React/Vue hooks and state. Understands component architecture. Can design the dashboard but might miss performance optimizations like virtualization.
- **Senior Level (Strong Hire):** Deep understanding of JS under the hood. Proactively brings up server-state caching (React Query), component reusability, lazy loading, and list virtualization during the architecture phase.
